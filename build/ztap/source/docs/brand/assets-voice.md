# OptiFinish Assets, Imagery, and Voice

This document defines the non-code brand expression rules that future teams should follow when creating new pages, campaigns, decks, or site extensions.

## 1. Brand Personality

OptiFinish should read as:

- advanced
- practical
- premium
- operator-aware
- deployment-ready

It should not read as:

- trendy startup-first
- corporate-manufacturing bland
- sci-fi fantasy
- AI buzzword-heavy
- hobby robotics

## 2. Voice Guidelines

### 2.1 Tone

Write with:

- clarity
- authority
- brevity
- specificity

### 2.2 Preferred Message Themes

- mimic skilled motion once
- store and recall coating intent
- convert manual expertise into repeatable execution
- reduce training burden
- shorten setup and changeover
- bring robotic coating to harder geometries

### 2.3 Preferred Writing Moves

- use short technical verbs
- mention line realities and operator realities
- turn features into operational consequences
- emphasize repeatability, recall, and deployment readiness

### 2.4 Avoid

- vague transformation language
- buzzwords like `revolutionary`, `disruptive`, or `AI-powered` unless there is a precise reason
- overly academic explanation
- copy that sounds written for investors instead of industrial buyers

## 3. Headline Rules

Headlines should be:

- short
- high contrast
- visually rhythmic
- easy to split across lines

Recommended structure:

- one core idea
- one highlighted yellow phrase
- one concrete completion phrase

Examples of the style already present:

- `Watch Z-TAP in action.`
- `Built for operators, not robot specialists.`
- `Come see Z-TAP coat your part.`

## 4. Body Copy Rules

- keep paragraphs compact
- make each sentence carry information
- use plain language where possible
- avoid long subordinate clauses
- describe what happens in sequence when process explanation is needed

## 5. CTA Copy Rules

Preferred CTA types:

- request
- visit
- watch
- download
- book

Examples:

- `Request Consultation`
- `Request Demo Access`
- `Download Spec Sheet`
- `Visit Our Lab`

Avoid:

- `Learn More`
- `Get Started`
- `Contact Us`

These are acceptable only if context makes the action specific elsewhere.

## 6. Imagery Direction

### 6.1 Preferred Subject Matter

- robotic spray/coating arms
- nozzles, guns, reciprocators, and fixtures
- workcells and controlled lab environments
- operator interaction with tooling
- UI or HMI displays
- technical close-ups with visible precision

### 6.2 Visual Mood

Imagery should feel:

- sharp
- grounded
- metallic
- controlled
- performance-oriented

### 6.3 Color Discipline

Images should naturally harmonize with:

- black
- warm gray
- metallic silver
- yellow
- soft white highlights

Avoid assets dominated by unrelated saturated colors.

## 7. Image Treatment

When placing imagery on the website:

- prefer large, confident crops
- spotlight the key machine or action
- frame within premium shells when the image is central to the section
- use subtle overlays to increase readability
- maintain strong subject contrast

For poster-style hero media:

- let the machine silhouette dominate
- use shadowing and glow to deepen focus
- keep labels minimal

## 8. Illustration And Diagram Style

If future pages require diagrams:

- use clean linework
- use yellow for active paths or key states
- keep background neutral
- prefer simple geometry over ornamental illustration
- ensure diagrams match the industrial precision tone

## 9. Iconography Rules

- simple geometry
- clean strokes
- modern technical feel
- limited use
- yellow reserved for emphasis

Preferred icon themes:

- motion path
- recognition
- recall
- deployment
- repeatability

## 10. Motion-As-Brand Guidance

Motion should visually suggest:

- scanning
- pathing
- alignment
- capture
- sweep
- docking
- activation

Preferred motion metaphors:

- progress fill
- guided reveal
- scanline
- measured lift
- magnetized attraction

Avoid:

- playful bounce
- random particle chaos
- flashy neon cyberpunk effects

## 11. Asset Packaging Recommendation

Future handoff package should include:

- logo originals and export variants
- favicon/app icon set
- approved imagery folder
- product device/mockup templates
- social share templates
- slide/deck cover templates
- icon and diagram starter kit

Suggested future structure:

- `/brand/logo`
- `/brand/imagery`
- `/brand/mockups`
- `/brand/icons`
- `/brand/social`
- `/brand/docs`
